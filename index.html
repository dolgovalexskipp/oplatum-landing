Один сервис — всё для приёма и роста

Работаем с цифровыми проектами в России и за её пределами: TG-боты, SaaS, образовательные продукты, маркетплейсы, иностранные юрлица с российской аудиторией. KYB проводит юрист команды лично — с пониманием контекста вашего бизнеса.

Для вашей команды разработки

Подключение — задача на день, не на неделю

API уровня Stripe: HMAC-подпись, идемпотентность, тестовая среда, вебхуки, готовый Node.js SDK. Сэкономит вашему разработчику 2–3 дня по сравнению с типовым российским эквайрингом.

step 01

Получите ключи

Apply for pilot → выдаём ak_test_ и ak_live_ в течение 1 рабочего дня.

step 02

Создайте платёж

POST /merchant-api/v1/payments/create-code с HMAC-подписью. Получите URL для QR или Checkout.

step 03

Слушайте webhook

Webhook с подписью HMAC-SHA256 → проверяете подпись, выдаёте товар/доступ. Резерв через Events API.

# 1. Подпишите запрос HMAC-SHA256
TIMESTAMP=$(date +%s)
NONCE=$(openssl rand -hex 16)
BODY='{"amount":1000,"walletType":1,"shopPaymentId":"order_42"}'
SIG=$(printf "%s.%s.POST./payments/create-code.%s" \
  $TIMESTAMP $NONCE $BODY | openssl dgst -sha256 -hmac $API_SECRET)

# 2. POST с подписью
curl -X POST https://oplatum.com/merchant-api/v1/payments/create-code \
  -H "ak: $API_KEY" \
  -H "timestamp: $TIMESTAMP" \
  -H "nonce: $NONCE" \
  -H "signature: $SIG" \
  -H "Idempotency-Key: order_42" \
  -d "$BODY"

import Oplatum from 'oplatum';

const oplatum = new Oplatum(process.env.OPLATUM_KEY);

const payment = await oplatum.payments.create({
  amount: 1000,
  walletType: 1,
  shopPaymentId: 'order_42',
}, { idempotencyKey: 'order_42' });

// СБП QR — payment.url для редиректа или картинки
res.redirect(payment.url);

import hmac, hashlib, time, secrets, requests, json

body = json.dumps({"amount": 1000, "walletType": 1, "shopPaymentId": "order_42"})
ts, nonce = str(int(time.time())), secrets.token_hex(16)
to_sign = f"{ts}.{nonce}.POST./payments/create-code.{body}"
sig = hmac.new(SECRET.encode(), to_sign.encode(), hashlib.sha256).hexdigest()

r = requests.post("https://oplatum.com/merchant-api/v1/payments/create-code",
  headers={
    "ak": API_KEY, "timestamp": ts, "nonce": nonce,
    "signature": sig, "Idempotency-Key": "order_42",
  }, data=body)

print(r.json()["url"])

Как подключиться

От заявки до первых клиентов — 7–14 дней

Каждым мерчантом занимаемся лично — от первого звонка до live-транзакции. Параллельно с проверкой документов ваша команда работает в тестовой среде.

01 · день 1
Заявка и звонок
Заполняете форму — в течение рабочего дня созваниваемся на 30 минут. Разбираем ваш кейс, договариваемся об условиях pilot.
02 · дни 1–5
KYB через живого человека
Присылаете стандартный пакет документов. Юристы ТОО «Clarion» и комплаенс НКО «МОБИ.Деньги» проверяют 3–5 рабочих дней.
03 · параллельно
Подключение API
Sandbox-доступ — в день заявки, не ждём KYB. Ваша команда подключает API, тестирует приём и подписки. Помогаем лично, если нужно.
04 · после KYB
Live и первые выплаты
Включаем приём боевых платежей. Settlement — по вашему запросу в USDT, USD или рублях. Первая выплата — на следующий рабочий день.

FAQ

Частые вопросы

Кто обрабатывает платёж? +

Покупатель платит через СБП. Приёмщик — НКО «МОБИ.Деньги», лицензированный ЦБ РФ оператор электронных денежных средств (лицензия №3523-К). Деньги поступают на счёт ТОО «Clarion» (Казахстан) — юрлицо Oplatum.

Как я получаю выплаты? +

Settlement в USDT, USD или рублях — выбираете при подписании. Pull-модель: запрашиваете выплату когда удобно. В режиме pilot programme — индивидуально по согласованию с командой.

Сколько занимает онбординг? +

Sandbox-ключи — в течение 1 рабочего дня после Apply. Первая test-транзакция — в тот же день. KYB и подписание в режиме pilot — 3–5 рабочих дней. Live-ключи — сразу после.

Какие SDK? +

Официальный Node.js SDK на fetch. Для Python и других языков — чистый REST с готовыми примерами. SDK не обязателен — большинство интегрируется напрямую через HTTP.

Рекурренты / подписки? +

Через СБП-подписку (walletType: 2). Покупатель один раз подтверждает — сервис списывает по расписанию произвольные суммы. У сервисов уровня Купикода в продакшне.

Что не делаете? +

Gambling / betting / adult / crypto-обменники — не работаем. Это политика. Карты МИР пока не на стороне приёма (только СБП). Физические товары с международной доставкой как MOR — нет.

А карты будут? +

На входе сейчас только СБП — это покрывает ~85% российской аудитории. Карты МИР обсуждаются с НКО МОБИ.Деньги, но они осторожны из-за высокого уровня скама в карточном эквайринге. Если карты критичны для вашего сценария — напишите, оценим конкретный кейс.

Какая комиссия? +

В режиме pilot programme — индивидуально по кейсу. После 2–3 первых клиентов будет публичная сетка. Apply for pilot и обсудим конкретные цифры.

А вы не закроетесь, как Freekassa? +

Лицензированный процессор + казахстанский юрконтур — две независимые точки устойчивости. Регулирование ЦБ означает прозрачность операций. Freekassa работала на кипрском SPV без лицензии — наша структура другая.
