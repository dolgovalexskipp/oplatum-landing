Платёжная инфраструктура для агентов, а не для людей

Классические эквайринги предполагают: форма, клик, ввод CVV. Для агентов нужна другая модель — агент действует от имени пользователя, в рамках заданных ограничений, с машинной атрибуцией каждого шага.

Intent-based API

Агент объявляет намерение (купить X за сумму Y у продавца Z), пользователь подтверждает одним действием, Oplatum исполняет. Никаких модальных окон и "введите CVV заново".

Токены с ограничениями

Пользователь выдаёт агенту кошелёк с лимитами — по сумме, продавцам, категориям, частоте. Токен отзывается одной кнопкой.

MCP-сервер

Готовый integration для Claude, OpenAI Agents SDK, Codex CLI и других LLM-платформ. Агент видит возможность оплатить как обычный инструмент.

Машинная атрибуция

Каждая транзакция привязана к агенту, пользователю, сессии и intent-цепочке. Прозрачная отчётность — кто, когда, за что заплатил.

Идемпотентность

Безопасные повторы, защита от дублей, машиночитаемые ошибки. Рассчитано на работу с LLM, которые могут «забыть» и повторить запрос.

Аудит и compliance

Полный аудит-лог действий агента, интеграция с KYC/AML-флоу, готовность к агентским регуляторным требованиям.
